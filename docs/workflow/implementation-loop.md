# 实现循环

AI 实现不应该是“一次生成到底”。更稳定的方式是短循环：理解、计划、实现、运行、修复、自检，然后交给独立 reviewer。

## 推荐循环

```text
1. 读规则和相关代码
2. 输出最小修改计划
3. 实现当前切片
4. 运行最小验证命令
5. 修复明确失败
6. 生成自检摘要
7. 交给独立 reviewer
```

## Implementer 的限制

- 不主动扩大范围。
- 不在测试失败时删除或弱化测试。
- 不新增长期兼容层，除非任务明确要求。
- 不把重构和行为变化混在一起。
- 不把事实源复制成第二份文档。

## 自检摘要

```markdown
# Implementer Self-Check

## 完成内容
- 

## 触达文件
- 

## 已运行验证
- 命令:
- 结果:

## 未运行验证
- 原因:

## 风险
- 

## 需要 reviewer 重点看的地方
- 
```

自检不是最终验收。它只是让 reviewer 更快定位影响面。
