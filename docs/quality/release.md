# 发布、灰度与回滚

AI 生成代码通过测试并不代表可以安全发布。发布验收关注真实环境的不确定性：配置、迁移、数据、监控、灰度和回滚。

## 发布前检查

| 项目 | 检查问题 |
| --- | --- |
| 配置 | 是否新增 env？默认值和示例是否同步？ |
| 数据 | 是否有 迁移？是否可回滚？是否影响生产数据？ |
| 兼容 | API / SDK / 类型是否同步？旧客户端是否可用？ |
| 监控 | 是否能看到成功率、错误率、延迟、业务指标？ |
| 灰度 | 是否有 feature flag、租户白名单或流量分层？ |
| 回滚 | 回滚命令、数据回滚、开关关闭是否明确？ |

## 发布审查 Agent 提示词

```text
你是发布审查 Agent。不要修改代码。

请审查当前 代码差异 的发布风险：配置、迁移、兼容、灰度、监控、回滚、文档和 运行手册。

输出：
- 阻断级/高风险。
- 必须补充的发布证据。
- 推荐灰度策略。
- 回滚步骤是否完整。
```

## 发布证据

AI 应在验收包中提供：

- 变更是否需要 feature flag。
- 生产配置是否需要更新。
- 迁移是否可重复执行。
- 新增指标、日志或告警。
- 回滚路径和风险。
