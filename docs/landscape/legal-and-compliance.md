# 法务、合规与版权边界

AI Coding 知识库和工程流程都要尊重版权、许可证、隐私与企业合规。AI 可以帮助总结和映射外部资料，但不能把第三方版权内容当成可自由搬运的素材库。

## 主要风险

| 风险 | 例子 | 控制 |
| --- | --- | --- |
| 版权搬运 | 逐字复制或翻译整篇商业文档 | 链接原文、短摘录、中文导读、章节映射 |
| 许可证污染 | 引入未知 license 代码片段 | dependency/license review、来源记录 |
| 隐私泄漏 | 把客户数据、日志、密钥 发给外部模型 | 脱敏、最小化、企业边界、审计 |
| 合规误判 | AI 自行解释法律义务 | 人类合规负责人裁决 |
| 数据驻留 | 代码、日志、PII 跨区域处理 | 供应商和区域策略 |

## 外部资料处理原则

- 可以整理：目录结构、核心观点、实践映射、执行清单、站内改造建议。
- 可以短引用：用于说明关键概念，但应克制。
- 不应做：整篇逐字搬运、整篇逐字翻译、绕过原站访问控制。
- 必须保留：原文链接、来源名称、更新时间或访问时间。

## AI 审查 Agent检查项

```text
1. 是否复制了过长的第三方原文？
2. 是否把外部代码片段引入生产代码且未记录 license？
3. 是否把用户数据、日志、密钥 放进提示词？
4. 是否需要法务/安全负责人裁决？
5. 是否提供了原文链接而不是替代原文？
```

## 预算不足时怎么做

合规问题不能靠“省 Token”跳过。预算不足时，应只做摘要和链接，不做长篇改写；涉及客户数据或版权争议时转人工裁决。
