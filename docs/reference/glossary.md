# 术语表

## AI Coding

以 AI Agent 参与实现、审查和验证的软件工程方式。重点不是代码补全，而是端到端交付系统。

## AI 原生验收

AI 先做多角色审查、测试质疑、安全红队和证据汇总，人类最后裁决高风险与争议项。

## Harness 引擎

围绕模型和上下文的工程运行层，负责工具、状态、权限、审查、证据、人工关口、发布和反馈学习。

## Token 预算

一次 AI Coding 任务可消耗的上下文、推理轮次、工具调用、CI 时间和人类注意力。预算不足时应缩小任务或压缩低风险环节，而不是跳过高风险验收。

## 简化 / 标准 / 严控模式

按风险和预算选择的三档执行模式。简化模式适合低风险小改动；标准模式适合普通功能；严控模式适合安全、数据、支付、租户、生产发布等高风险变更。

## MCP

Model Context Protocol。用于把工具、资源和提示词暴露给模型客户端的协议，适合连接文档、代码、CI、浏览器、数据库只读查询和内部工具。

## A2A

Agent-to-Agent 协议。用于让多个独立 Agent 以对等方式协作，适合跨系统、跨角色的长任务协调。

## 持久执行

长任务在失败、中断、重启后可以从检查点恢复，而不是依赖聊天历史继续。

## 人工关口

流程中显式的人类决策节点，通常用于高风险、不可逆、争议、产品判断和发布批准。

## 实现 Agent

负责实现当前切片的 AI。它可以修改代码和运行验证，但不应该给自己最终验收结论。

## 审查 Agent 组

多个独立 AI 审查 Agent 的集合。每个审查 Agent聚焦一种风险，例如需求、测试、安全、治理、发布。

## 测试质疑 Agent

专门审查测试有效性的 Agent。它的目标是找出测试为什么可能不能证明正确性。

## 安全红队 Agent

专门寻找可利用攻击路径、安全配置错误和权限绕过的 Agent。

## 汇总裁决 Agent

聚合审查结果的 Agent。它只去重、定级、标出冲突和人类决策点，不新增事实。

## 验收证据包

将需求、代码差异、测试、链路记录、截图、日志、问题项、风险和回滚路径汇总成的人类裁决材料。

## 人工裁决者

最终负责风险接受、产品判断、架构方向和发布决策的人类负责人。
