# 度量指标

AI Coding 的指标不能只看“写了多少代码”。代码量变多不等于交付质量提高。更重要的是吞吐、返工、缺陷、审查噪音和风险发现效率。

## 推荐指标

| 指标 | 含义 | 目标 |
| --- | --- | --- |
| 交付周期 | 从需求到合并的时间 | 缩短 |
| 审查等待时间 | 从 AI 完成到人工裁决的时间 | 缩短 |
| AI 返工率 | AI 初次实现后的返工次数 | 下降 |
| 人工介入次数 | 人工介入次数 | 聚焦高价值 |
| 有效问题占比 | AI 审查发现的有效问题比例 | 提高 |
| 逃逸缺陷数 | 合并后才暴露的问题 | 下降 |
| 测试质疑命中率 | 测试审查发现的有效缺口 | 初期上升，后期稳定 |
| 风险豁免数 | 被接受但未立即修复的风险 | 受控且有过期时间 |

## 噪音控制

AI 审查如果输出太多低价值评论，会让团队失去信任。建议：

- 默认只在 PR 上发布阻断级、高风险和需要人工裁决的问题。
- 中低风险建议放在折叠报告或 CI 产物里。
- 对误报做反馈学习。
- 每周删除或改写低信号规则。
