# 角色与职责

AI Coding 团队不一定需要新增岗位，但需要明确角色职责。一个人可以承担多个角色，但责任边界不能模糊。

| 角色 | 职责 | 不负责 |
| --- | --- | --- |
| Product Owner | 定义目标、非目标、验收标准 | 逐行审代码 |
| Engineering Owner | 架构方向、风险接受、最终合并 | 手工跑所有低层检查 |
| AI Implementer | 实现小切片、修复明确问题 | 自己批准自己 |
| AI Reviewer | 只读审查、输出证据 | 修改代码或做最终裁决 |
| AI Judge | 聚合 finding、定级、去重 | 发明新事实 |
| Security Owner | 接受或拒绝安全风险 | 依赖普通 reviewer 顺带看安全 |
| Release Owner | 灰度、监控、回滚、发布批准 | 忽略运行时证据 |

## 人类 Owner 的核心动作

- 批准任务切片。
- 批准高风险修复策略。
- 裁决 AI reviewer 冲突。
- 接受或拒绝风险豁免。
- 对最终合并和发布负责。
