# 角色与职责

AI Coding 团队不一定需要新增岗位，但必须明确责任边界。一个人可以承担多个角色，但不能让 AI 或工具替代最终负责人。

| 角色 | 职责 | 不负责 |
| --- | --- | --- |
| 产品负责人 | 定义目标、非目标、验收标准 | 逐行审代码 |
| 工程负责人 | 架构方向、风险接受、最终合并 | 手工跑所有低层检查 |
| AI 实现 Agent | 实现小切片、修复明确问题 | 自己批准自己 |
| AI 审查 Agent | 只读审查、输出证据 | 修改代码或做最终裁决 |
| 汇总裁决 Agent | 聚合问题、定级、去重 | 发明新事实 |
| 安全负责人 | 接受或拒绝安全风险 | 依赖普通审查顺带看安全 |
| 发布负责人 | 灰度、监控、回滚、发布批准 | 忽略运行时证据 |

## 负责人的核心动作

- 批准任务切片。
- 批准高风险修复策略。
- 裁决 AI 审查冲突。
- 接受或拒绝风险豁免。
- 对最终合并和发布负责。
