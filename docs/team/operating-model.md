# 组织运行模型

AI Coding 不是个人效率工具升级，而是团队交付系统升级。组织需要重新定义谁写规则、谁审风险、谁维护事实源、谁拥有最终责任。

## 推荐运行方式

```text
工程师定义目标和边界
AI Implementer 完成小切片
AI Reviewer Swarm 先审
AI Judge 汇总风险
人类 Owner 裁决
发布系统验证真实环境
反馈进入规则和 Prompt
```

## 三条组织原则

- **规则前置**：不要靠口头提醒 AI；把规则写进仓库事实源。
- **审查自动化**：人类不做第一层筛查，先让 AI 过滤明显问题。
- **责任不外包**：AI 可以生成建议，最终 owner 仍由人类承担。

## 周期性维护

| 周期 | 动作 |
| --- | --- |
| 每个 PR | AI review、测试审查、风险摘要 |
| 每周 | 汇总高频 AI 错误，更新规则和 Prompt |
| 每月 | 复盘 AI 产能、缺陷、返工、审查噪音 |
| 每季度 | 清理 compat/deprecated/dead，更新工具链 |
