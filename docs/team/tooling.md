# 工具选型

AI Coding 工具应按职责选择，不要期待一个工具覆盖全部流程。关键是让工具之间通过事实源、结构化输出和 CI 连接起来。

## 工具层次

| 层次 | 代表能力 | 选择标准 |
| --- | --- | --- |
| 本地编码 Agent | 读写代码、运行命令 | 是否理解仓库、是否可控 |
| PR 审查 Agent | 自动审代码差异、发评论 | 信噪比、规则支持、上下文能力 |
| 安全审查 Agent | 威胁建模、攻击路径验证 | 是否能复现或给证据 |
| 浏览器 Agent | 页面验证、截图、链路记录 | 是否能覆盖真实用户路径 |
| CI 编排 | 自动运行审查和验证 | 权限隔离、日志可追踪 |
| 知识库 | 规则、ADR、运行手册、提示词 | 是否容易维护和引用 |

## 选型问题

- 是否支持仓库级规则文件？
- 是否能限制审查 Agent 只读？
- 是否能输出结构化 JSON？
- 是否能接入 PR 评论和 CI 产物？
- 是否能处理单仓多项目和跨文件上下文？
- 是否能控制权限并防提示词注入？
- 是否能学习团队反馈，而不是制造更多噪音？

## 最小工具组合

```text
本地编码 Agent
+ PR AI 审查 Agent
+ 测试质疑 Agent 提示词
+ 安全审查 Agent 提示词
+ 浏览器链路记录 / 截图
+ CI 自动汇总报告
```
